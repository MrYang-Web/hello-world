15.全局作用域

       作用域:函数的作用范围
             function fun() {
               var a =123;
             }
             alert(a);
      此时并不会执行弹窗,会提示 a is undefind


         作用域一共分为两种:
         1.全局作用域
               -直接编写在script标签中的代码,都在全局作用域
               -function之外,script之内,就是全局作用域
               -全局作用域在页面打开时创建,在页面关闭时销毁.
               -在全局作用域中,有一个全局对象window,
                           它代表着我们的浏览器窗口,它由浏览器创建.
                           我们可以直接使用.
               -在全局作用域中,我们创建的变量都会作为window对象的属性保存.
               -在全局作用域中,们创建的函数都会作为window方法的属性保存.
               -在页面的任意地方,都可以看得到.
                   var a = 10;
                   console.log(window.a);
        那么我们全局作用域下创建一个函数,那么也就是给window创建了一个方法.
        function fun1() {
          console.log("hello");
        }
        window.fun1();
        根据这个原理  window.alert("hello"); === alert("hello");

        ---------变量声明提升:
        01.   var a =123; 前面不写var  一样可以赋值给变量a, 就是相当于window.a=123

                  console.log(a);
                    var a =123;
        此时如果把var a =123;写在 console.log(a);下面,是打印不出123的.

        ------变量的声明提升-------
        原因就是使用var关键字声明的变量,会在所有的代码执行之前声明.(但是不会被赋值)
        相当于在所有script代码的最上面写了一句     var a;
        然后再在下面对 a  进行赋值.
        不写var 的话,变量就不会被声明提升.

        -----------------------------------------------------------------------
          fun2();
          fun3();

          函数声明,会被提前创建
          function fun2() {
            console.log("这是函数二");
          }

          函数表达式,不会被提前创建,但是var fun3会被提前声明(但是不会赋值)所
          以代码在61行之前, fun3 == undefind;
          var fun3 = function () {
            console.log("这是函数三");
          }
          这里fun2();能被执行         fun3();会出现报错
          ---函数声明提升-----
          使用函数声明形式创建的函数   function functionName() {  };
          它会在所有代码执行之前就被创建.所以我们可以在函数声明前来调用.
